inputs¶

The number of channel inputs which scopeio is going to plot.

input_preamp¶

This parameter is required to set the analog inputs when to have different volatge resolution. If all the inputs have the same resolution, set it to (0 to inputs-1 => 1.0)

layout_id¶

layout_id selects one of the two display layouts. The table below shows the parameter values to be set, according to the video resolution required.

Value Resolution Video frequency
0 1920x1080 150 MHz
1 800x600 40 MHz

There is a nano-window system in which other layouts can be described pretty easily. So far, there are only two.

vt_div¶

It represents the vertical base division. The least five significant bits represent the binary point. The default value b”0_001_00000” means 1.00000. See vt_scales for more explanation.

hz_div¶

It represents the horizontal base division. The least five significant bits represent the binary point. The default value b”0_001_00000” means 1.00000. See hz_scales for more explanation.

hz_scales¶

hz_scales is sixteen-elements-long vector whose elements are scale_t records. Each one describes one of the the sixteen horizontal scales using hz_div as a base to display the corresponding values on the screen. The steps to set up each element of hz_scales are the followings:

A
Choose your sample rate: in the exmaple it is 800 KS/s
B
Each division has 32 pixels, The base division is gotten by dividing 32 by the sample rate. In the example the result is 40 us as the sample rate is 0.8 MS/s
C
Set the record member step to 40.00. step only controls the increment of the horizontal axis marks.
D
Get the corresponding ascii code of the factor character and set it to deca. In the example: the corresponding factor is micro.
E

The hz_div parameter is composed of five fraction bits, three integer bits and one sign bit. Three integer bits mean that integer part of hz_div must be one digit only. The horizontal base division must be aligned according to that.

Following the example: The result of the horizontal base division is 40.00. To fit it in hz_div, the decimal point should be shifted one position to the left to get the one-digit-only integer part. The new value, then, is 4.000, and the corresponding binary representation is 0_100_00000 in which the first ‘_’ charater separates the sign bit and second one, the fraction bits.

F
Once the correct value is selected for hz_div, the record member scale should be set, according to the scale table, to display the correct horizontal base division value on the screen. The scale member is a four bit vector whose two left bits shift the decimal point while the other two bits on the right select a number from: 1.0, 2.5, 5.0 or 2.0 by which the hz_div is multiplied. The proper number is selected by combining all of the four bits.
G
Set the record member mult according to the required downsampling. if the downsampling is going to be made by multiplexing the input channel of ADCs set it to 1. Use tdiv to know which scale has been selected by the user.
H
Check the examples.
